# ScreenPrivacy

[English](../README.md) 🇬🇧 | [Italiano](README.it.md) 🇮🇹 | [Español](README.es.md) 🇪🇸 | [Français](README.fr.md) 🇫🇷 | [Deutsch](README.de.md) 🇩🇪 | [Русский](README.ru.md) 🇷🇺

ScreenPrivacy предоставляет легкий экран приватности для SwiftUI, который автоматически закрывает конфиденциальные представления, когда приложение становится неактивным, а обнаружение записи или захвата экрана выступает дополнительным опциональным уровнем. Он также включает безопасный рендеринг по умолчанию, который предотвращает снимки экрана и записи защищенного контента. Это решение рассчитано на однострочное подключение и сохраняет UI чистым, тестируемым и уважительным к приватности.

Полезные случаи использования:
- Вы показываете балансы, медицинские данные или личную информацию.
- Вы поддерживаете многозадачность и хотите безопасные снимки приложения.
- Вам нужен быстрый экран по умолчанию, но с полной настройкой.
- Вы хотите блокировать снимки и записи без дополнительной настройки.

## Требования

- iOS 17.0 или новее
- Swift 6.0 или новее

## Установка

Добавьте ScreenPrivacy как зависимость Swift Package в Xcode или через `Package.swift`.

## Использование

Примените модификатор к любому представлению, которое хотите защитить:

```swift
import ScreenPrivacy
import SwiftUI

struct AccountView: View {
    var body: some View {
        AccountDetailsView()
            .screenPrivacyShield()
    }
}
```

Предоставьте кастомный экран, если хотите фирменный стиль:

```swift
import ScreenPrivacy
import SwiftUI

struct AccountView: View {
    var body: some View {
        AccountDetailsView()
            .screenPrivacyShield {
                VStack(spacing: 12) {
                    Image(systemName: "lock.shield")
                        .symbolRenderingMode(.hierarchical)
                        .imageScale(.large)
                        .font(.largeTitle)
                    Text("Приватно")
                        .font(.title2)
                        .bold()
                }
                .frame(maxWidth: .infinity, maxHeight: .infinity)
                .background(.background)
                .foregroundStyle(.primary)
            }
    }
}
```

Используйте контейнер, если предпочитаете композицию:

```swift
import ScreenPrivacy
import SwiftUI

struct AccountView: View {
    var body: some View {
        ScreenPrivacyContainer {
            AccountDetailsView()
        }
    }
}
```

## Поведение

- Показывает экран, когда сцена становится неактивной.
- Может дополнительно включать экран при обнаружении захвата экрана.
- Применяет `privacySensitive()` к защищенному контенту.
- Использует безопасный рендеринг по умолчанию, чтобы блокировать снимки и записи.
- Безопасный рендеринг реализован через размещение SwiftUI внутри canvas безопасного текстового поля.

## Настройка

- Установите `isEnabled` в `false`, чтобы отключить экран.
- Установите `includeCaptureDetection` в `false`, если нужен экран только при неактивности.
- Установите `blocksScreenCapture` в `false`, если нужен только экран без безопасного рендеринга.

## Советы

- Используйте минимальный кастомный экран для плавных переходов.
- Предпочитайте `.background(.background)` для адаптивного светлого/темного вида.
- Если используете свой цвет фона, убедитесь, что текст читаем в обеих темах.

## FAQ

**Блокирует ли это снимки экрана?**  
Да, по умолчанию. ScreenPrivacy использует контейнер на основе безопасного текстового поля, когда `blocksScreenCapture` равен `true`, а это значение по умолчанию.

**Работает ли это в виджетах или расширениях?**  
Этот пакет предназначен для SwiftUI-представлений в приложении. Он не предназначен для защиты таймлайнов виджетов.

**Можно ли добавить аналитику или логирование?**  
Да. Вы можете обернуть `ScreenPrivacyContainer` и наблюдать за жизненным циклом приложения для логирования событий, не меняя экран.
